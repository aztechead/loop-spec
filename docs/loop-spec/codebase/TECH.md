## Runtime Requirements (as of v1.0.0)

The agent-teams execution mode adds hard requirements beyond the tools listed above:

| Requirement | Detail |
|---|---|
| **Claude Code >= v2.1.32** | Minimum version for agent teams. CC **>= 2.1.178** removed `TeamCreate`/`TeamDelete`; `lib/teams-capability.sh` version-gates the cycle onto the implicit-team model (`Agent({name})`) on those builds and the legacy explicit model below it. Teams are optional — absent them, the no-teams fallback runs. |
| **`CLAUDE_CODE_EXPERIMENTAL_AGENT_TEAMS=1`** | Opt-in flag for agent teams. Set in the environment (or in `.claude/settings.json` under `env`) before launching `claude`. When unset the cycle records `teamsMode=none` and uses one-shot `Agent` dispatch; it does not abort. Override the resolved mode with `LOOP_SPEC_TEAMS_MODE=none\|explicit\|implicit`. |
| **Agent teams harness capabilities** | The cycle skill runs a capability probe verifying: TeamCreate, TaskUpdate status, TaskUpdate metadata, SendMessage, concurrent-claim. If any check fails, the skill aborts. See `docs/loop-spec/PREREQUISITES.md` for the full probe spec. |

## Plugin / Integration Points

| Integration | Details |
|---|---|
| **Claude Code plugin system** | Registered via `.claude-plugin/plugin.json` (version 1.0.0). Skills in `skills/*/SKILL.md`, slash commands in `commands/*.md`, agent definitions in `agents/loop-spec-*.md`, hooks in `hooks/hooks.json`. |
| **Claude Code hooks** | `hooks/hooks.json` registers `hooks/restrict-agent-paths.sh` as a `PreToolUse` hook matching `Write\|Edit` tool calls. Hook parses the session transcript JSONL to determine caller subagent type and enforces path-scoped write restrictions. Agent-teams event hooks live in `hooks/team/`: `task-created.sh`, `task-completed.sh`, `teammate-idle.sh`. |
| **Anthropic model API (via claude CLI)** | Three models consumed at runtime: `claude-opus-4-7`, `claude-sonnet-4-6`, `claude-haiku-4-5`. Model selection is governed by `skills/shared/preset-matrix.md`. |
| **GSD (get-shit-done)** | Optional ingest path. If the target project has `.planning/codebase/`, `lib/gsd-ingest.sh` concatenates GSD artifacts into loop-spec format at first-run setup. No GSD dependency at dev time. |
| **git** | Worktree-based parallel execution (EXECUTE phase creates per-task worktrees under `.loop-spec/worktrees/{slug}/task-NNN/`), feature branches (`feat/{slug}`), atomic state with SHA tracking. |

## External Services
